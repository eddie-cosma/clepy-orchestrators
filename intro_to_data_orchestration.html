The Big Picture: Data Pipelines

  • Ingest → [Transform ⇆ Store] → Analyze
  • Orchestration coordinates each step and manages dependencies

ETL vs. ELT

Source: https://www.simform.com/blog/data-pipeline/

  • ETL was the traditional approach—transform early because compute resources were limited in warehouses.
  • ELT emerged with cloud data warehouses that make it easy to store raw data and transform later.
  • Orchestrators like Airflow and Dagster can handle either approach.

What is Data Orchestration?

  • Coordinating execution, dependencies, and state of data tasks
  • Scheduling, dependency management, retries, monitoring

Why Use an Orchestrator?

  • Reliability
  • Reproducibility
  • Scalability
  • Observability
  • Collaboration

What is a DAG?

  • DAG = Directed Acyclic Graph
  • Represents tasks (nodes) and dependencies (edges) in a workflow
  • Directed: edges have a direction, indicating execution order
  • Acyclic: no cycles allowed — you can’t loop back to a previous task
  • Ensures workflows have a clear start and end

Further reading: https://en.wikipedia.org/wiki/Directed_acyclic_graph

DAGs are the backbone of most modern orchestrators like Airflow and Dagster. The "Directed" part means each step clearly leads to the next. "Acyclic" prevents infinite loops — you can branch and merge paths, but execution always moves forward. This concept allows us to visualize and reason about complex workflows.

Demo: Airflow


from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime

def fetch_data():
    print("Fetching data...")

with DAG("etl_airflow", start_date=datetime(2023, 1, 1), schedule_interval="@daily") as dag:
    fetch_task = PythonOperator(task_id="fetch", python_callable=fetch_data)
